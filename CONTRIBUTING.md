# Contribuir

Gracias por cuidar esta herramienta como un espacio comun.

## Buenas contribuciones

- Cambios pequenos, legibles y faciles de probar.
- Presets visuales con nombres claros.
- Mejoras de accesibilidad y rendimiento.
- Documentacion para usos pedagogicos, performaticos o comunitarios.
- Ideas para mantener el proyecto local-first y abierto.

## Antes de cambiar mucho

Abre un issue con:

- que quieres hacer;
- para quien sirve;
- como se usaria en una sesion real;
- si agrega dependencias nuevas.

## Criterios

- Evitar dependencias cerradas.
- Preferir tecnologias open source.
- No subir archivos del usuario a servidores.
- Mantener la interfaz simple y manipulable en vivo.
